页面 + 功能 + 字段总表

这是一份适合直接交给开发或外包的基础说明页。它把当前网站中最关键的页面、每页要实现的功能、以及核心字段整理到一起，方便后续开发、排期和沟通。

使用方式

这份总表不是最终数据库文档，而是产品阶段最重要的一份开发沟通底稿。先用它明确每个页面到底做什么、需要收集哪些信息，再进入技术实现和数据库设计会更高效。

建议你下一步把这里继续拆成两份更细的材料：一份给产品/自己看，一份给开发看。

核心页面总表

页面 页面作用 核心功能 关键字段 优先级
首页
index.html
 平台主入口，对外展示品牌、入口和平台价值。 导航跳转、推荐商家展示、分类入口、品牌入口。 推荐商家名称、城市、材料标签、跳转链接。 必做
找商家
merchant-list.html
 浏览和筛选商家列表。 商家列表、筛选、排序、跳转商家详情。 商家名称、城市、打印类型、材料、评分、起印价格、认证状态。 必做
商家详情
merchant-detail.html
 展示商家能力、案例和服务范围。 查看案例、查看设备能力、发起询价/提交需求。 商家名称、简介、设备型号、支持材料、案例图、评分、服务说明。 必做
发布需求
request-print.html
 用户提交打印需求。 表单提交、上传文件、跳转成功页。 姓名/昵称、联系方式、模型文件、材料、数量、精度、交期、备注。 必做
商家入驻
merchant-join.html
 商家提交入驻申请。 表单提交、资料上传、跳转成功页。 店铺名/姓名、联系方式、城市、设备型号、支持材料、案例图、服务说明。 必做
登录 / 注册
login.html
 用户与商家进入账号系统。 登录、注册、身份区分、验证码入口。 手机号/邮箱、密码、验证码、用户类型。 必做
用户中心
user-center.html
 用户查看自己的需求、订单和收藏。 查看需求、查看订单、查看收藏、进入详情。 用户昵称、需求列表、订单状态、收藏商家、评价状态。 必做
商家后台
merchant-dashboard.html
 商家管理询价、订单和店铺资料。 查看询价、报价、更新状态、维护案例。 店铺资料、询价列表、订单状态、案例列表、设备信息。 必做
管理后台
admin-dashboard.html
 平台审核商家、管理订单与内容。 审核、下架、查看投诉、处理订单争议。 商家资料、审核状态、订单编号、举报内容、处理记录。 必做
帮助中心
help-center.html
 整理常见问题和服务说明。 FAQ 分类、问题展示、跳转规则页。 问题标题、问题分类、答案内容。 建议做
联系我们
contact-us.html
 提供商务合作、支持咨询入口。 联系方式展示、留言表单。 姓名/公司、联系方式、咨询类型、留言内容。 建议做
意见反馈
feedback.html
 收集产品建议和错误反馈。 反馈表单、类型选择、跳转成功页。 姓名/昵称、联系方式、反馈类型、反馈内容。 建议做
成功反馈页体系
request-success / join-success / feedback-success
 表单提交后的成功状态页。 显示成功状态、下一步建议、跳转下一页。 状态文案、按钮链接、说明文本。 建议做
规则 / 协议 / 隐私
platform-rules / privacy-policy / terms-of-service
 承载平台基础规则与合规说明。 内容展示、互相跳转、页脚入口。 章节标题、条款内容、更新时间。 建议做

第一阶段最关键的字段

模块 建议字段 说明
用户账户 用户ID、昵称、手机号/邮箱、密码、用户类型、注册时间 先支持最基础的用户体系，区分普通用户和商家。
商家资料 商家ID、店铺名、联系人、城市、设备型号、支持材料、案例图、审核状态 这是平台供给侧最关键的数据。
打印需求 需求ID、用户ID、文件、材料、数量、精度、交期、备注、状态 这是用户侧最核心的提交对象。
订单信息 订单ID、用户ID、商家ID、需求ID、报价、状态、创建时间、完成时间 后续所有流转和评价都会围绕订单展开。
评价信息 评价ID、订单ID、评分、评价内容、创建时间 用于沉淀平台信任感。

交给开发时的建议说法

你可以直接把这一页发给开发，并补一句：

“我希望先按这个表做 MVP 第一阶段，先实现账号、商家入驻、发布需求、后台审核、订单状态和基础评价，其他复杂功能先不做。”

这样开发会更容易理解你的项目边界，也更容易给你评估工期和报价。
