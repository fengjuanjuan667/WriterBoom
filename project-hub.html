创业资料中心 / 项目资料页

这一页把 WriterBoom 目前所有和“项目表达、产品规划、开发协作”相关的资料统一收口。以后你对外介绍项目、找开发、找合作、做路演，或者自己整理思路，都可以从这里进入。
建议把这一页视为项目总入口：先讲项目是什么，再讲怎么做，最后讲怎么落地。

项目表达

项目介绍 / 商业模式
适合对外说明这个项目为什么成立、服务谁、未来怎么赚钱。
关于我们
适合表达平台定位、价值主张和阶段目标。
品牌规范
适合统一 Logo、品牌色、字体和组件规范。

产品规划

MVP 功能规划
适合明确第一阶段做什么、第二阶段再做什么。
页面 + 功能 + 字段总表
适合直接给开发看，是最实用的一页。
站点地图
适合快速查看现在整个网站已经有哪些页面。

开发协作

开发协作 / 技术实现建议
适合和开发、外包、技术合伙人沟通实施方式。
平台规则
适合开发理解平台需要承载的审核、售后和处罚逻辑。
用户协议
适合明确平台、用户、商家之间的基础关系。

推荐阅读顺序

第 1 步：先看项目介绍 / 商业模式
先明确项目是什么、解决什么问题、为什么值得做。
第 2 步：再看 MVP 功能规划
先确定第一阶段只做哪些功能，不要无限扩张。
第 3 步：最后看页面 + 功能 + 字段总表
这是最适合进入开发沟通的具体材料。

这页最适合发给谁

开发 外包团队 合作伙伴 技术朋友 自己未来复盘

以后你只需要把这一个页面发出去，对方就能快速找到项目介绍、功能规划、开发建议和字段总表，而不用在很多页面里来回翻。
